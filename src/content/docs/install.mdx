Browy comes in two parts:

1. **The browser extension**: a Chromium side panel plus a DevTools panel.
2. **The native host**: a small Node.js helper that runs on your
   machine and talks to GitHub Copilot.

Install the extension from the Chrome Web Store, then run the
one-line installer for the native host.

## Requirements

<CardGrid>
  <Card title="Operating system" icon="laptop">

## Step 1: install the extension

<LinkCard
  title="Add Browy to Chrome / Edge / Brave"
  href="https://chromewebstore.google.com/detail/iondecjdokngnlkfpipgolgkfegpmjca"
  description="One click on the Chrome Web Store. Same listing covers Chrome, Edge, and Brave."
/>

Pin the Browy toolbar icon so the side panel is one click away.

<Aside type="tip" title="✓ Validate">
Click the Browy toolbar icon. The side panel should open and show a
"host disconnected" banner. That is expected until Step 2.
</Aside>

<Aside type="tip" title="Prefer to sideload?">
The native-host installer below also stages an unpacked copy of the
extension at `~/Downloads/Browy-Extension/` (or
`%USERPROFILE%\Downloads\Browy-Extension\` on Windows). Open
`chrome://extensions/`, toggle **Developer mode** on, click **Load
unpacked**, and select that folder. The native host pre-approves both
the Web Store and unpacked extension IDs, so you can switch between
them without re-running anything.
</Aside>

## Step 2: install the native host

<Tabs syncKey="os">
<TabItem label="Windows" icon="seti:windows">

Open PowerShell and run:

```powershell
irm https://github.com/BrowyHQ/browy/releases/latest/download/install.ps1 | iex
```

The installer will:

<Steps>
1. Download the latest Browy release (~130 MB)
2. Install the native host to `%LOCALAPPDATA%\Browy\app\`
3. Register `com.browy.host` with Chrome / Edge / Brave (pre-approves both the Web Store and unpacked extension IDs)
4. Stage an unpacked extension copy at `%USERPROFILE%\Downloads\Browy-Extension\` for sideload installs
</Steps>

<Aside type="tip" title="✓ Validate">
Re-open the side panel. The "host disconnected" banner should be gone.
If not, run `& "$env:LOCALAPPDATA\Browy\app\browy.exe" --version` to
confirm the binary is on disk.
</Aside>

</TabItem>
<TabItem label="macOS" icon="seti:apple">

Open Terminal and run:

```bash
curl -fsSL https://github.com/BrowyHQ/browy/releases/latest/download/install.sh | bash
```

The installer will:

<Steps>
1. Detect your architecture (Apple Silicon or Intel) and download the matching tarball (~125 MB)
2. Install the native host to `~/.browy/app/`
3. Register `com.browy.host` with Chrome / Edge / Brave (pre-approves both the Web Store and unpacked extension IDs)
4. Stage an unpacked extension copy at `~/Downloads/Browy-Extension/` for sideload installs
</Steps>

<Aside type="tip" title="✓ Validate">
Re-open the side panel. The "host disconnected" banner should be gone.
If not, run `~/.browy/app/browy --version` to confirm the binary is on
disk.
</Aside>

</TabItem>
<TabItem label="Linux" icon="linux">

Open a terminal and run:

```bash
curl -fsSL https://github.com/BrowyHQ/browy/releases/latest/download/install.sh | bash
```

The installer will:

<Steps>
1. Detect your architecture (`x86_64` or `aarch64`) and download the matching tarball (~133 MB)
2. Install the native host to `~/.browy/app/`
3. Register `com.browy.host` with Chrome / Edge / Brave (writes to `~/.config/<browser>/NativeMessagingHosts/`, pre-approves both extension IDs)
4. Stage an unpacked extension copy at `~/Downloads/Browy-Extension/` for sideload installs
</Steps>

<Aside type="tip" title="✓ Validate">
Re-open the side panel. The "host disconnected" banner should be gone.
If not, run `~/.browy/app/browy --version` to confirm the binary is on
disk.
</Aside>

</TabItem>
</Tabs>

## Step 3: sign in to Copilot

Click the Browy icon to open the side panel. The first time you use it,
Browy will prompt you to sign in.

<figure class="browy-frame">
  <Image src={panelSignin} alt="The Browy side panel with the 'Sign in to GitHub Copilot' button visible" />
  <figcaption>Click "Sign in to GitHub Copilot" and follow the device-flow link.</figcaption>
</figure>

A terminal window opens with the GitHub Copilot **device-flow link**.
Paste the code into the URL it shows, authorize Browy in your browser,
and the terminal closes itself.

<Aside type="tip" title="✓ Validate">
Send "what is the headline of this page" against any tab. You should
see a `snapshot` tool call followed by a streamed reply.
</Aside>

You're done. Try [your first chat](/first-chat/) for five concrete examples.

place.

## Uninstall

<Tabs syncKey="os">
<TabItem label="Windows" icon="seti:windows">

<Steps>
1. Remove the extension from `chrome://extensions/`
2. Delete `%LOCALAPPDATA%\Browy\` and `%USERPROFILE%\.browy\`
3. Delete the native messaging host registration:
   `%LOCALAPPDATA%\Google\Chrome\User Data\NativeMessagingHosts\com.browy.host.json`
   (and the equivalent under Edge / Brave folders)
</Steps>

</TabItem>
<TabItem label="macOS" icon="seti:apple">

<Steps>
1. Remove the extension from `chrome://extensions/`
2. Run the bundled uninstaller, then remove the install directory:
   ```bash
   ~/.browy/app/uninstall.sh && rm -rf ~/.browy ~/Downloads/Browy-Extension
   ```
3. The uninstaller deletes the native messaging host manifests under
   `~/Library/Application Support/Google/Chrome/NativeMessagingHosts/`
   (and the equivalent for Edge / Brave).
</Steps>

</TabItem>
<TabItem label="Linux" icon="linux">

<Steps>
1. Remove the extension from `chrome://extensions/`
2. Run the bundled uninstaller, then remove the install directory:
   ```bash
   ~/.browy/app/uninstall.sh && rm -rf ~/.browy ~/Downloads/Browy-Extension
   ```
3. The uninstaller deletes the native messaging host manifests under
   `~/.config/google-chrome/NativeMessagingHosts/` (and the equivalent
   for Edge / Brave).
</Steps>

</TabItem>
</Tabs>

## Troubleshooting

<Aside type="caution" title="“Host disconnected, reconnecting…”">
The side panel can't reach the native host. Common causes: the native
host failed to install (re-run the installer), the browser was opened
before the host was registered (restart the browser), or your AV /
endpoint protection quarantined the bundled `node` (allow the install
directory, `%LOCALAPPDATA%\Browy\` on Windows, `~/.browy/` on macOS
and Linux, in your AV exclusions).
</Aside>

<Aside type="caution" title="“Host unreachable, reinstall →”">
The reconnect attempts gave up. Click **Reinstall host** in the
banner; it points at the same one-line installer you used the first
time (`install.ps1` on Windows, `install.sh` on macOS / Linux). Your
chats and settings are preserved across reinstalls.
</Aside>

<Aside type="note" title="“Browy started debugging this browser”">
Chrome's standard debugging banner appears while a session is active.
That's how Browy reads and drives pages. Closing the banner detaches
the agent. See [Permissions → debugger](/permissions/) for the
full story.
</Aside>
